It's 2024, I should build a blog!

After a long time thinking about it, I finally decided to go ahead and build my own blog.

It seems only fitting to use Drupal as it has been my main bread-winner and the CMS I am most comfortable with. Once I started revisiting my forged-by-experience and strongly-held beliefs on how Drupal should be used to build your own blogging platform, I realised that my opinions were going to be truly challenged. A lot of what I will be writing about in this blog is supposed to be a logbook of that journey.

This takes us to my first bike-shedding moment: I'm cheap and I don't want to pay for hosting. So, after a bit of research I landed on the following option: Github Pages. It's 2024 and everyone and their cat has a Github account (I'll save my full opinion on it for another article). This means you can use Github Pages as a free hosting service. Great!

So, after reading about it, turns out it only allows static hosting - ooofff. Unfortunately, this would mean that it won't pair up with Drupal, as it is a full-fledged PHP-based framework, therefore, it requires hosting with PHP support and a database service. That's not what I want.
 

Enter... Tome!

So this got me thinking: is there any way of making Drupal static?

Turns out there is: Tome. This is a Drupal community contributed module that promises to take your Drupal site and generate a full static version of it. There are some obvious caveats: the editorial experience only exists locally, there's no backend-powered search or dynamic paging and filtering in views).

This means, however, that with some careful content architecture planning and if you are happy with having the editorial experience available to you on your own local environment, this can actually work.

The proof it does work is that this blog is built with Drupal, made static with Tome and is currently hosted on Github Pages. Yay, me!

What next?

Well, now that I feel pretty good about myself for building this blog, some custom Drupal stuff, doing some light theming, designing a logo and favicon shenanigans, writing this article and statically hosting it in less than a day's worth of work - what's next?

In the next few weeks I will post some articles with a more in-depth review of all the steps that got me here, as well as some ideas and improvements I will be doing to this site/blog.

Watch this space!
